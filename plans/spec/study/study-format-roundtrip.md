# Spec: study-format-roundtrip

## 한 줄 정의
사용자가 Study 프로필을 저장하고 다시 열면 덱, 카드, 학습 진행률, 복습 일정, 성취 기록이 손실 없이 복원된다.

## 진입점
- 자동: 학습/복습 세션 완료 시 자동 저장으로 데이터가 기록된다.
- 메뉴: File > Save / Open으로 프로필 데이터를 저장/불러온다.

## 사용자 흐름
1. 사용자가 덱을 만들고 카드를 추가한다.
2. 학습 세션에서 카드를 학습하고 평가한다.
3. 복습 일정에 따라 간격 반복 학습을 수행한다.
4. 저장 시 모든 학습 데이터가 로컬 스토리지에 기록된다.
5. 사용자가 앱을 닫았다가 다시 연다.
6. 프로필을 열면 덱, 카드, 학습 진행률, 복습 일정이 그대로 복원된다.

## 성공 조건 (Acceptance Criteria)
- [ ] 덱 구조와 카드 내용(앞면, 뒷면, 이미지, 수식)이 손실 없이 복원된다.
- [ ] 각 카드의 학습 이력(평가 기록, 난이도 계수, 다음 복습일)이 유지된다.
- [ ] 오늘의 복습 대기열이 정확하게 재계산된다.
- [ ] 성취 배지, 연속 학습 일수(streak), 총 학습 시간이 유지된다.
- [ ] 커리큘럼 진행률이 유지된다.
- [ ] 학습 통계(일별/주별/월별 학습량)가 유지된다.
- [ ] 프로필 설정(학습 시간 알림, 일일 목표)이 유지된다.
- [ ] 마지막으로 학습한 덱과 카드 위치가 복원된다.

## 실패 / 취소 흐름
- 저장 중 오류가 발생하면 이전 상태가 보존된다.
- 손상된 프로필 파일을 열면 에러 메시지가 표시된다.
- 디스크 공간이 부족하면 저장 실패 메시지가 표시된다.

## 경계 / 예외
- 대규모 덱(1,000+ 카드)도 합리적인 시간 내에 열린다.
- 학습 이력이 많아도(10,000+ 평가 기록) 대기열 계산이 즉시 수행된다.
- 같은 카드가 여러 덱에 공유되어도 독립적인 학습 진행률이 유지된다.

## 범위 외
- 클라우드 동기화는 별도 spec으로 분리.
- 덱 공유/내보내기는 별도 spec으로 분리.
