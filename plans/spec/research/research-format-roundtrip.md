# Spec: research-format-roundtrip

## 한 줄 정의
사용자가 Research 라이브러리를 저장하고 다시 열면 논문, 첨부파일, 메타데이터, 주석, 태그가 손실 없이 복원된다.

## 진입점
- 자동: 편집/추가/삭제 시 자동 저장으로 라이브러리 데이터가 기록된다.
- 메뉴: File > Save / Open으로 라이브러리 파일을 저장/불러온다.
- 드래그 앤 드롭: PDF 파일을 드롭하여 라이브러리에 추가한다.

## 사용자 흐름
1. 사용자가 논문을 추가, 수정, 삭제한다.
2. 주석, 하이라이트, 태그, 컬렉션 구조를 편집한다.
3. 저장 시 모든 데이터가 로컬 스토리지에 기록된다.
4. 사용자가 앱을 닫았다가 다시 연다.
5. 라이브러리를 열면 모든 논문, 첨부파일, 주석, 태그, 컬렉션 구조가 그대로 복원된다.

## 성공 조건 (Acceptance Criteria)
- [ ] 논문 메타데이터(제목, 저자, 연도, 학회/저널, DOI)가 손실 없이 복원된다.
- [ ] PDF 첨부파일이 그대로 접근 가능하다.
- [ ] PDF 위의 하이라이트, 밑줄, 취소선, 스티키 노트가 원래 위치에 복원된다.
- [ ] 컬렉션 트리 구조(폴더 계층)가 유지된다.
- [ ] 태그와 태그-논문 연결이 유지된다.
- [ ] 즐겨찾기 상태가 유지된다.
- [ ] BibTeX/RIS 가져오기 데이터가 정확하게 복원된다.
- [ ] Q&A 항목과 연결된 논문 페이지가 유지된다.
- [ ] 마지막으로 열었던 논문과 페이지 위치가 복원된다.

## 실패 / 취소 흐름
- 첨부 PDF 파일이 이동/삭제된 경우 "첨부파일을 찾을 수 없습니다" 경고가 표시된다.
- 손상된 라이브러리 파일을 열면 에러 메시지와 함께 빈 라이브러리로 시작할지 묻는다.
- 저장 중 오류가 발생하면 이전 상태가 보존된다.

## 경계 / 예외
- 대규모 라이브러리(10,000+ 논문)도 합리적인 시간 내에 열린다.
- PDF 주석 데이터가 메타데이터와 분리되어 저장되어도 복원 시 올바르게 매핑된다.
- 같은 논문이 여러 컬렉션에 속해 있어도 중복 없이 유지된다.

## 범위 외
- BibTeX/RIS 내보내기는 별도 spec으로 분리.
- 클라우드 동기화는 별도 spec으로 분리.
- DOI 자동 조회는 별도 spec으로 분리.
